\chapter{Implementation}
\label{ch:implementation}



Converting biological complexity into working code requires more than good programming --- it demands architectural vision. This chapter reveals how moving from Mesa to Repast4Py enabled true 3D simulation and MPI parallelism, transforming a proof-of-concept into a production-scale virtual tumor.

This chapter examines the architectural decisions, design patterns, and implementation strategies that transform biological theory into executable software. Rather than cataloging code structure, we focus on the engineering problems encountered and the design patterns employed to solve them, revealing how computational constraints shape biological model design and vice versa.

\section{Spatial Index Design}
\label{sec:spatial-index}

Repast4Py lacks built-in neighbor queries, so the model maintains a manual spatial index using defaultdict(list) that maps coordinates to agent lists. Grid utilities provide 3D neighborhood computation using Moore or Von Neumann neighborhoods with boundary checking.

\section{Agent Serialization}
\label{sec:agent-serialization}

Repast4Py agents require (id, type, rank) construction and maintain uid tuples for MPI distribution. The AgentType IntEnum maps 18 cell types to integer IDs, and each agent implements a save() method returning (uid, state_tuple) for serialization. Type identification uses integer comparison of uid[1] rather than isinstance() checks.

\section{Glucose Field Implementation}
\label{sec:glucose-field-impl}

The GlucoseField class uses NumPy 3D arrays and SciPy convolution for performance and control. Diffusion uses a 6-connected discrete Laplacian kernel with zero-flux boundary conditions via mode='nearest'. The implementation avoids Repast4Py's SharedValueLayer for simplicity and debugging convenience.

\section{Entry Point and Execution}
\label{sec:entry-point}

The simulation runs via `mpirun -n 1 python run.py` using Repast4Py's schedule runner to execute the main step() loop with data collection, termination checking, effect propagation, treatment, glucose updates, agent stepping, and deferred movement.
